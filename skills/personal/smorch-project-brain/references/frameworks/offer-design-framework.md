# Offer Design Framework — Hormozi Grand Slam Offer + Value Stack

## PURPOSE
Use this when generating offer.md. Transforms a feature list into a compelling offer with clear value framing, risk reversal, and commercial grounding.

---

## THE HORMOZI VALUE EQUATION

Value = (Dream Outcome × Perceived Likelihood of Achievement) / (Time Delay × Effort & Sacrifice)

**To increase value, you can:**
1. Increase Dream Outcome: make the result bigger
2. Increase Perceived Likelihood: add proof, guarantees, credentials
3. Decrease Time Delay: deliver results faster
4. Decrease Effort: reduce what the customer has to do

**For the brain file, document all 4 levers for this specific offer.**

---

## VALUE STACK CONSTRUCTION

The value stack is the complete list of what the customer gets, each with standalone value.

### Step 1: List Every Deliverable
Not features, DELIVERABLES. What does the customer walk away with?

| Deliverable Type | Example | Notes |
|-----------------|---------|-------|
| Core service/product | "Signal detection engine monitoring 100 target accounts" | The main thing they're buying |
| Tools/templates | "Pre-built Clay enrichment waterfall for MENA" | Saves them from building it |
| Training/onboarding | "2-hour setup workshop with your team" | Reduces time-to-value |
| Access | "Private WhatsApp group with other signal-based GTM teams" | Network value |
| Support | "Weekly 30-min optimization call for 3 months" | Reduces perceived risk |
| Bonuses | "Competitor signal benchmark report" | Sweetens the deal |

### Step 2: Assign Standalone Value to Each
"If you bought this separately, it would cost $X"

**Rules:**
- Values must be credible. Don't claim a template is "worth $5,000" if similar ones exist for free
- Use market comparables: "A consultant would charge $200/hour for this, 10 hours = $2,000"
- The sum of standalone values should be 3-10x the actual price (the "value gap")

### Step 3: Stack Value Against Price
Total standalone value: $X
Your price: $Y
Value gap: X/Y ratio

**Healthy ratios by project type:**
- Tech product: 5-10x (monthly savings should be 5-10x monthly price)
- Consulting: 3-5x (deliverable value 3-5x engagement fee)
- Training: 10-20x (lifetime earnings impact far exceeds course price)
- Agency: 3-5x (pipeline value 3-5x monthly retainer)

## PRICING FRAMEWORKS

### Consulting / Agency
- **Value-based:** Price = % of expected outcome (e.g., "We charge 10% of the pipeline we generate")
- **Retainer:** Fixed monthly fee for defined scope. Most common for agency work
- **Project-based:** Fixed fee for defined deliverable. Clear scope, clear end date
- **Hybrid:** Small retainer + performance bonus. Aligns incentives

### Tech Product
- **Per-seat:** Scales with team size. Works when value is per-user
- **Usage-based:** Scales with consumption. Works when value correlates to usage
- **Flat-rate tiers:** Simple, predictable. Works when value doesn't vary much by usage
- **Freemium:** Free tier + paid upgrade. Only works with viral/self-serve product

### Training
- **Founding price:** Discounted launch price for early adopters (creates urgency + testimonials)
- **Standard price:** Full price post-launch
- **Premium/VIP:** Includes additional access (1-on-1 calls, private group, extended support)
- **Community upgrade:** Monthly membership after initial course (recurring revenue)

### Pricing Context Rule
**offer.md must ALWAYS include at least one competitor's price for anchoring.** A price without context is meaningless. "$497/month" means nothing. "$497/month vs. Competitor X at $1,200/month for fewer features" means everything.

## GUARANTEE / RISK REVERSAL

The guarantee directly addresses the buyer's #1 objection.

| Guarantee Type | When to Use | Example |
|---------------|-------------|---------|
| **Money-back** | Low-cost products, first-time buyers, high trust needed | "Full refund within 30 days if you don't see results" |
| **Performance** | Agency/consulting where you control the outcome | "15 qualified meetings in 90 days or we work for free until you get them" |
| **Conditional** | When results depend on client execution too | "Complete all 5 modules + implement the system. If you don't see ROI in 60 days, full refund" |
| **Anti-guarantee** | Premium positioning, scarcity | "No guarantee. We select 5 companies per quarter. Our track record speaks for itself" |
| **Hybrid** | High-ticket consulting | "50% milestone-based: first payment on kickoff, second on delivery of agreed metrics" |

**Rule:** If no guarantee exists, document WHY. "No guarantee because [specific reason]" is better than leaving it blank.

## DELIVERY FRAMEWORK

### Time-to-Value
How quickly does the customer get their first result? Document explicitly.

**Tiers:**
- Immediate: Value on day 1 (tool access, template delivery)
- Quick win: First meaningful result in 1-2 weeks
- Full outcome: Complete transformation in [X weeks/months]

### Client Effort Matrix
| Activity | Client Does | We Do | Frequency |
|----------|-----------|-------|-----------|
| [Activity 1] | [their part] | [our part] | [how often] |
| [Activity 2] | | | |

This prevents scope creep and sets realistic expectations.

---

## OFFER VALIDATION CHECKLIST

After generating offer.md, verify:

- [ ] Value stack has 3+ components, each with standalone value
- [ ] Total standalone value is 3-10x the actual price
- [ ] Price includes competitive context (at least 1 competitor's price)
- [ ] Guarantee/risk reversal is specified (or explicitly noted as "none" with reasoning)
- [ ] Delivery timeline is specific (not "ongoing")
- [ ] Client effort is documented (what they need to provide/do)
- [ ] Time-to-first-value is explicit
- [ ] Success metrics are quantified (the ONE number that proves it worked)
- [ ] ROI framing is present (customer's cost of problem vs. price of solution)
