- **The context files (project brain) are living documents.** Update them as projects evolve. Outdated context produces outdated output

## 4. Quality Standards

The bar is high. Every output represents SMOrchestra.

### Universal Standards (Everyone)
- Reply same day. No exceptions
- Bookings from campaigns need to be attended and qualified
- Campaigns must run on time. Delays are not acceptable
- Linear self-reporting is mandatory. If it's not in Linear, it didn't happen

### Razan-Specific
- Videos must have high standards, no errors. Quality is non-negotiable for all training content and commercial campaigns
- Campaign for commercials must execute on time with correct positioning

### Nour-Specific
- LinkedIn posts must have high standards. Writing that reads like AI-generated copy will not be tolerated
- Every post must sound like a human with 20 years of MENA experience wrote it
- If you don't fully understand what you wrote, don't post it. Generic AI-sounding content damages credibility

### Lana-Specific
- Code delivery on time. Deadlines are commitments
- No losing code due to no backup. Version control at all times
- Follow the AI-native process. No shortcuts on the 5-layer architecture
- Ability to roll back at all times. No excuse for losing track of version control

## 5. Accountability Rules

### The Core Rule
**Unaccountability will not be tolerated for any reason.**

### Transparency Rule
If there is a screwup, Mamoun needs to know about it. He should not discover problems on his own. The cost of hiding a mistake is always higher than the cost of reporting it.

**What "report immediately" means:**
- Campaign didn't launch on time: report within the hour
- Video has errors that shipped: report before it reaches audience
- LinkedIn post went out with wrong messaging: report immediately
- Code lost, build broken, deployment failed: report within the hour
- Client-facing mistake: report before the client notices

### What Gets Escalated vs. What Gets Handled
- **Handle yourself:** Typos in internal docs, minor delays under 24 hours, tool configuration issues, routine troubleshooting
- **Escalate to Mamoun:** Client-facing errors, missed deadlines on deliverables, quality failures on shipped content, lost code or data, campaign execution failures, any decision that changes scope or budget

## 7. Reporting Cadence

- **Daily:** Linear task updates (self-reported by each team member)
- **Weekly:** Status summary per owned project (what shipped, what's blocked, what's next)
- **Immediately:** Any screwup, missed deadline, quality failure, or client-facing issue
