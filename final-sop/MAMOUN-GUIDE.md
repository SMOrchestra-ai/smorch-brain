## What's Running Automatically

### Hooks (Every Claude Code Session, Both Servers)

| Hook | When It Fires | What It Does |
|------|--------------|--------------|
| SessionStart | Session opens | Reminds agent to read skill manifest + lifecycle triggers |
| PreToolUse: git commit | Before any commit | Reminds to run `/review` first (SOP-02 quality gate) |
| PostToolUse: git push/PR | After push or PR creation | Reminds to run `/deploy-checklist` (SOP-03) |
| PreToolUse: destructive | Before rm -rf, force push, etc. | **BLOCKS** the command, asks for approval |
| PreToolUse: SQL guard | Before DROP/TRUNCATE | **BLOCKS** destructive SQL |
| PreToolUse: secret scanner | Before writing files | **BLOCKS** if API keys detected in code |
| PostCompact | After context compaction | Restores skill + project context |

### n8n Workflows (smo-dev — flows.smorchestra.ai)

| Workflow | Schedule | What You Get |
|----------|----------|-------------|
| Weekly Stakeholder Update | **Sunday 9am Dubai** | Telegram message: active issues, by status, by assignee, top priorities |
| Tech Debt Quarterly Review | **First Monday of Jan/Apr/Jul/Oct** | Linear ticket created with tech debt scan across all repos |
| Linear Issue Done → Summary | **Instant** (webhook) | Telegram message when any Linear issue moves to Done |
| BRD → Auto-PRD Pipeline | **On demand** (webhook) | POST a BRD to `/webhook/brd-to-prd`, get back a PRD auto-filed in Linear |

## What Needs Your Approval

### Codex Doctrine (SOP-11)
Sulaiman or al-Jazari will send you a **Workforce Plan** before starting any project:

```
This project: 6 agents for 9 tasks
- Codex agents: Task 1 (high-volume data migration) + Task 4 (bulk test generation)
- Claude Code: Tasks 2,3,5,6,7,8,9 (architecture, UI, integrations)
Reasoning: [why each assignment]
```

**You:** Approve, adjust, or reject. They don't dispatch until you say go.

### Decision Authority Matrix (What Lana Escalates to You)
- Database schema migrations
- New third-party integrations (cost implications)
- Production deployments
- Supabase RLS policy changes
- Architecture decisions
- Credit system / pricing logic changes
- Security-sensitive changes (webhook signatures, auth)

### Everything Else
Agents and Lana handle independently. You only get pinged for the above.

## How to Use the BRD → PRD Pipeline

```bash
curl -X POST https://flows.smorchestra.ai/webhook/brd-to-prd \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Real-time Lead Scoring Dashboard",
    "project": "SSE V3",
    "brd": "Problem: Sales team cannot see lead scores updating in real-time...\nSolution: Live dashboard with WebSocket updates...\nScope: MVP for UAE market first..."
  }'
```

Returns: PRD auto-generated by Gemini Flash, filed as a Linear issue.
