## Observations

1. **smorch-gtm-engine is the most drifted plugin.** 4 skills have source files 2.7x-4.8x larger than deployed. The deployed Cowork instance is running old/truncated versions of asset-factory, signal-detector, signal-to-trust-gtm, and wedge-generator. These are core GTM skills, so this is a revenue-impacting gap.

2. **Two skills are larger in deployed than source.** content-systems (mamoun-personal-branding) and smorch-about-me (smorch-context-brain) were likely updated in Cowork but never pulled back to smorch-brain. Risk: next push from source will overwrite those improvements.

3. **Version tracking is broken.** Only 1 of 9 deployed plugins retains a plugin.json with version info. This makes version-based drift detection impossible for the other 8.

## Recommended Actions

### Urgent (do today)
1. **smorch-gtm-engine:** Re-push the plugin from smorch-brain to Cowork. 4 core skills are running outdated versions that are 60-80% smaller than source.
2. **Reverse-sync deployed improvements:** Before pushing, diff the deployed versions of `content-systems` and `smorch-about-me` against source. If deployed has genuine improvements, pull those into smorch-brain first.

### This week
3. **Fix version tracking:** Ensure all plugin pushes include plugin.json in the deployed directory. Without this, version comparison is blind.
4. **Audit minor drifts:** The 13 minor mismatches (<10%) across mamoun-personal-branding and smorch-gtm-engine suggest incremental edits happening in one location without syncing. Decide on a sync cadence.

5. **Backup directories in source:** smorch-brain contains `eo-microsaas-os-backup-v1.0.0` and `smorch-gtm-engine-backup-v1.0.0`. Consider moving these to a separate archive to keep the plugins directory clean.
